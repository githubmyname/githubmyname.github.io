基于Windows的Hexo博客搭建

一、准备工作

1、安装Git

Git是一个开源的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理。

记得勾选新建Git Bash

2、安装Node.js

npm安装hexo

选择下载的时候：LTS为长期支持版本，Current为当前最新版本，我选择了LTS版的安装

配置环境变量：在系统变量的Path里添加git的安装路径（没有添加path会报错：command not found)

桌面任意位置右键git bash

node -v显示版本信息则安装成功

二、安装Hexo轻量化博客框架

新建一个文件夹存储搭建博客所需的配置文件，例如D:\Blog，打开Blog文件夹右键Git Bash Here安装Hexo框架

2、新建一篇博客

四、部署到GitHub/Coding上

1、什么是GitHub

我们可以看一下百度百科怎么说：GitHub是一个面向开源及私有软件项目的托管平台，因为只支持git 作为唯一的版本库格式进行托管，故名GitHub。

2、Coding

可以看作是中国的GitHub，因其服务器在国内，访问起来比较快而且稳定，而且语言为中文相较于GitHub的全英文页面比较友好。

3、这篇博客是部署在GitHub上的所以我就先以部署在GitHub上为例

  • 创建GitHub账号链接

  • 新建一个repository，名称为yourname.github.io,其中yourname是刚刚创建的GitHub的名称

  • 填写好信息点击创建

  • 创建SSH

    在gitbash执行

    一路回车，根据gitbash给出的路径找到id_rsa.pub用记事本打开全选复制到粘贴版

    回到GitHub的Settings->SSH and GPG Keys->New SSH Key把id_rsa.pub里的Key粘贴上，title随便取，如blog。

  • 在gitbash 中验证是否添加成功

  • 修改本地_config.yml文件的配置

  • 部署到GitHub

    安装hexo-deployr-git上传

    清空本地缓存文件重新生成静态文件然后部署

    （之前我一这在这里卡了好久，原因是git版本太低，所以如果到了这一步deploy失败，试试更新git到最新版本）

    deploy成功后就可以到https://yourname.github.io查看啦

hexo博客插入本地图片不显示

这几天刚刚搭好我的个人博客，想要上传内容却发现有不少问题，比如这个插入本地图片页面不显示的问题，百度了一下发现这个问题还是挺常见的，但是它的解决办法并不适用于我这次遇到的问题，所以觉得有必要记录下来以便自己以后再遇见这样的问题回过头来看这篇博客能有所启发。

问题描述：

MarkDown编辑器Typora插入图片时路径显示为电脑上的绝对路径，例如（F:\blog\img\smile.jpg)，在博客所在文件夹下右键gitbash运行命令行hexo g(hexo generate),hexo s(server)之后在loalhost:4000预览本地页面发现图片无法显示，ok,百度一下：链接

hexo博客的搭建-笔记

本博客主要根据win教程搭建而成win+hexo部署则参考学长的教程Hexo博客搭建Linux+hexo

一、环境搭建

1、git

2、Node.js

3、hexo

二、本地建站

1、例如新建文件夹F:\blog\xxx.github.io，并在此目录下右键gitbash进入命令行界面

2、安装hexo：npm install -g hexo-cli

3、淘宝镜像：npm config set registry “https://registry.npm.taobao.org"

4、初始化：hexo init

npm install

5、本地预览：hexo server(或者hexo s),在浏览器地址栏输入localhost:4000访问博客默认页面

markdown写法请参考：官方文档

三、部署到github/Coding上

1、github/Coding：代码托管仓库，可以免费提供公开域名以便分享自己的博客

2、建议使用Coding，因为它的服务器位于国内，一般来说无论访问速度还是上传速度都优于github

3、 安装插件：npm install hexo-deployer-git –save

注册/登录coding，新建项目，blog

修改_config.yml

修改coding地址：

![](2018-08-20-about\coding.png)

4、保存，hexo deploy

5、开启coding的静态page服务，部署来源选择master分支，保存后自动运行，通过访问地址给他人分享你的博客
